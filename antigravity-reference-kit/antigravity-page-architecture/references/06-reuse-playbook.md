# Reuse Playbook

## What To Transfer

Transfer:

- system discipline
- section sequencing
- motion hierarchy
- audience segmentation
- ecosystem thinking

Do not transfer:

- proprietary branding
- exact copy
- exact assets
- exact visual signature

## Apply To A Serious Client Site

### For a B2B service site

Translate the pattern into:

- strong short hero
- early proof
- capabilities grid
- sector or audience segmentation
- credibility layer
- clear CTA close

### For a SaaS

Translate the pattern into:

- hero with demo
- modular product explanation
- use cases
- pricing
- changelog or release notes
- docs entry points

### For an agency

Translate the pattern into:

- short premium positioning
- proof or case preview
- process
- service architecture
- verticals served
- contact or booking CTA

## Delivery Checklist

### System

- Define color tokens.
- Define spacing tokens.
- Define type scale.
- Define grid.
- Define radii.
- Define easing set.

### Hero

- Keep the headline short.
- Add a primary CTA.
- Add a secondary CTA.
- Add proof within one or two scrolls.

### Motion

- Limit the motion vocabulary.
- Reuse the same durations and curves.
- Tie animation to hierarchy.

### Content

- Separate promise from explanation.
- Separate explanation from proof.
- Segment by audience.
- close with a clear action.

### Product Depth

- If the project is serious, create more than a home.
- Add at least a minimal interior ecosystem.

## Reusable Prompt Starters

- `Use $antigravity-page-architecture to turn this generic landing into a premium product-style page without copying Antigravity branding.`
- `Use $antigravity-page-architecture to design a concise hero, proof section, capabilities system, audience segmentation, and CTA plan for this client.`
- `Use $antigravity-page-architecture to derive tokens, motion rules, and section sequencing for this redesign.`
