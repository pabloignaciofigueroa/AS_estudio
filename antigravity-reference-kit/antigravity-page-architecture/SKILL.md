# Antigravity Page Architecture

## Overview

Use this skill to translate the structural strengths of Antigravity into other projects.
Use it to plan page architecture, section sequencing, visual systems, motion rules, conversion flows, and ecosystem depth for serious websites.

## Workflow

Follow this order unless the user request is narrower.

1. Identify the project type.
2. Identify the required seriousness level.
3. Translate Antigravity principles into that context.
4. Build or critique the page as a system, not as isolated sections.
5. Keep legal and creative boundaries clear.

## Project Type Mapping

Choose the closest mode before making recommendations.

- For `SaaS` or product marketing sites:
  prioritize hero clarity, demo proof, feature architecture, use cases, pricing, docs, changelog, and product depth.
- For `B2B services`:
  prioritize authority, proof, process, capabilities, industry segmentation, and contact conversion.
- For `Agencies`:
  prioritize premium positioning, work proof, methodology, service architecture, and selective motion.
- For `Enterprise-facing pages`:
  prioritize trust, navigation depth, system rigor, and multiple decision-maker paths.

## Core Transfer Rules

Apply these rules in every use.

- Copy principles, not appearance.
- Keep the hero short.
- Show proof before long explanation.
- Use a real design system with tokens.
- Use motion to clarify hierarchy, not to decorate.
- Segment by audience when the product or client needs it.
- Build route depth or at least the perception of system depth for serious projects.
- Repeat CTA with different context across the page.

## Output Expectations

When using this skill, produce outputs that feel actionable.

- If the user asks for strategy, return section architecture, system rules, and rationale.
- If the user asks for design direction, return tokens, typography, spacing, motion families, and component logic.
- If the user asks for implementation, turn the principles into concrete page sections, components, CSS variables, and interaction rules.
- If the user asks for critique, focus first on hierarchy, proof, route depth, and system coherence.

## Safeguards

Always protect these boundaries.

- Do not copy Antigravity branding, exact copy, logos, or proprietary-looking assets.
- Do not recommend ornamental complexity without narrative purpose.
- Do not overload the hero with explanation.
- Do not leave the page as a single isolated landing if the brief requires seriousness and depth.
- Do not use motion that harms readability or performance.
